Comprehensive System Requirements

1. Core Platform & User Management

Authentication: Users must be able to create an account and log in securely.

Tickers: The system must maintain a master list of all tradeable US equities and ETFs.

API Efficiency: The system must strictly manage marketdata.app API limits (10k/day) by relying heavily on short-TTL caching and batch background processing.

2. The Options Strategy Engine (Max 4 Legs)

Hard Limit: No strategy can exceed 4 option legs.

Strategy Templates (NEW): Instead of guessing what a user built, the system uses templates (e.g., "Single Call", "Vertical Spread", "Iron Condor").

Leg Validation: Based on the template, the UI will enforce the leg setup (e.g., a Vertical Spread requires exactly 2 legs of the same type, different strikes, same expiration).

Local OCC Generation: All OCC symbols must be generated by our local C# algorithm, zero API calls used for formatting.

3. Watchlists & Live Data

Multiple Watchlists: A user can have multiple named watchlists (e.g., "Tech Earnings", "Dividend Plays").

Background Polling: A C# background worker will poll the API every 10 seconds only for the OCC symbols currently saved in active watchlists.

Live Push: The UI will receive live updates via SignalR (WebSockets) without the user refreshing the page.

4. Price Alerts (NEW FEATURE)

Functionality: Users can set target prices on their saved strategies. (e.g., "Alert me if my AAPL Iron Condor net credit drops below $1.50").

5. Simulated Paper Trading (NEW FEATURE)

Functionality: Users can "Execute" a strategy from their watchlist. It doesn't use real money, but it saves the current live price to a Trade_History table.

Redis Key-Space Map (How we plan NoSQL)

Since we don't use ER diagrams for Redis, we map out the "Keyspace" to plan our ephemeral memory.

1. Option Chain Structure (TTL: 12 Hours)

Key Format: Chain:{Ticker}:{DTE} (e.g., Chain:AAPL:20230726)

Value: JSON Array of strikes and types available.

Purpose: Powers the UI dropdowns without hitting the database or API constantly.

2. Live Market Quotes (TTL: 10 Seconds)

Key Format: Quote:{OCC_Symbol} (e.g., Quote:AAPL230726C00200000)

Value: JSON Object { Bid, Ask, Delta, Theta, UnderlyingPrice, Timestamp }

Purpose: The Single Source of Truth for live prices. The background worker writes here; the WebSocket reads from here.

3. Active Symbols Set (TTL: Persistent while app runs)

Key Format: Worker:ActiveSymbols

Value: Redis SET containing all unique OCC symbols currently across all user watchlists.

Purpose: Tells the background worker exactly which symbols to query in its batch API call.
